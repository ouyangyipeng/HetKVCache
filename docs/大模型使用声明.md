# 大模型使用声明

## 声明摘要

在HetKVCache项目开发过程中，我们使用了大型语言模型（LLM）作为辅助工具。本文档详细说明了大模型在项目中的具体使用情况、使用范围和成果归属。

---

## 一、使用的大模型信息

### 1.1 使用的模型

| 模型名称 | 提供商 | 使用场景 | 使用时间 |
|---------|-------|---------|---------|
| Claude | Anthropic | 代码开发、文档编写 | 2024年3月-4月 |
| GPT-4 | OpenAI | 技术调研、方案讨论 | 2024年3月 |

### 1.2 使用方式

大模型主要通过以下方式辅助项目开发：

1. **代码编写辅助**
   - 提供代码框架和结构建议
   - 协助解决编译错误和调试问题
   - 提供算法实现参考

2. **文档撰写辅助**
   - 提供文档结构和内容建议
   - 协助技术文档的编写和润色
   - 提供中英文翻译支持

3. **技术调研辅助**
   - 协助分析相关技术方案
   - 提供技术选型建议
   - 协助理解复杂概念

## 三、人工核心贡献

### 3.1 核心创新点

以下核心创新点完全由人工完成：

1. **三级存储层次架构设计**
   - HOT/WARM/COLD层级划分
   - 层级间迁移策略
   - 存储介质特性利用

2. **热度评估算法**
   - 多维度加权评估公式
   - 频率、新近度、层重要性的权重配置
   - 阈值策略的调优

3. **无碎片分配策略**
   - 固定大小块设计
   - 内存池管理方案
   - 碎片彻底消除方案

4. **自动化评测系统**
   - 评测指标设计
   - 工作负载模拟
   - 对比分析方案

### 3.2 关键决策

以下关键决策由人工完成：

1. 技术栈选择（C++17 + CUDA）
2. 模块划分和接口设计
3. 性能优化策略
4. 测试方案设计

### 3.3 质量把控

以下工作由人工完成：

1. 代码审查和重构
2. 编译错误修复
3. 性能测试和调优
4. 文档审核和修订

## 四、大模型使用规范

### 4.1 使用原则

1. **辅助原则**：大模型仅作为辅助工具，核心创新和决策由人工完成
2. **验证原则**：大模型生成的所有代码和文档都经过人工验证和修改
3. **声明原则**：如实声明大模型的使用情况和贡献程度
4. **责任原则**：项目质量由人工负责，大模型不承担任何责任

### 4.2 使用限制

1. 不使用大模型直接生成最终代码，需经人工审核修改
2. 不使用大模型替代人工进行关键决策
3. 不隐瞒大模型的使用情况和贡献程度
