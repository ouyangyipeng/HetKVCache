# HetKVCache 方案设计文档

## 一、项目概述

### 1.1 项目背景

随着大语言模型(LLM)的快速发展，推理服务对计算资源的需求急剧增长。在LLM推理过程中，KV Cache（Key-Value Cache）是注意力机制的核心组件，其管理效率直接影响推理性能和成本。传统的KV Cache管理方案存在以下问题：

1. **内存利用率低**：固定分配策略导致内存碎片化严重
2. **访问模式单一**：缺乏对不同访问热度数据的差异化处理
3. **迁移策略简单**：无法充分利用异构存储层级优势
4. **缺乏自动化评估**：难以快速验证优化效果

### 1.2 项目目标

HetKVCache旨在构建一个高效、智能的异构存储管理系统，实现以下目标：

- **性能目标**：相比基线方案提升30%以上的吞吐量
- **效率目标**：内存利用率提升至90%以上
- **延迟目标**：P99延迟降低20%以上
- **易用性目标**：提供一键式评测脚本，支持主流推理框架

### 1.3 技术路线

本项目采用"监控-决策-执行"的闭环架构，包含四大核心模块：

## 二、核心模块设计

### 2.1 监控与热度评估器 (Monitor & Profiler)

#### 2.1.1 功能描述

监控模块负责实时追踪KV Cache块的访问模式，通过多维度指标评估数据热度，为迁移决策提供数据支撑。

#### 2.1.2 热度评估算法

采用加权综合评分法，热度分数计算公式：

```
heat_score = α × frequency_score + β × recency_score + γ × layer_importance_score
```

其中：
- **frequency_score**：访问频率得分，采用指数衰减模型
  ```
  frequency_score = 1 - exp(-λ × access_count)
  ```
  
- **recency_score**：访问新近度得分，采用时间衰减函数
  ```
  recency_score = exp(-Δt / τ)
  ```
  其中 Δt 为距离上次访问的时间间隔，τ 为衰减时间常数

- **layer_importance_score**：层重要性得分，基于Transformer层级特征
  - 浅层（0-1/3层）：权重 1.0
  - 中层（1/3-2/3层）：权重 0.8
  - 深层（2/3-1层）：权重 0.6

#### 2.1.3 分层阈值策略

| 层级 | 热度分数范围 | 存储位置 | 访问延迟 |
|------|-------------|----------|----------|
| HOT | [0.7, 1.0] | GPU VRAM | < 1μs |
| WARM | [0.3, 0.7) | Host RAM | 1-10μs |
| COLD | [0.0, 0.3) | NVMe SSD | 10-100μs |

#### 2.1.4 关键数据结构

```cpp
// 访问记录结构
```

### 2.2 迁移引擎 (Migration Engine)

#### 2.2.1 功能描述

迁移引擎负责执行数据在不同存储层级间的移动，包括：
- 基于热度评分的迁移决策
- 异步数据传输
- 迁移队列管理
- 预取优化

#### 2.2.2 迁移决策算法

采用代价-收益模型，迁移决策考虑以下因素：

```
migration_benefit = (target_tier_latency - current_tier_latency) × access_probability
migration_cost = data_size / transfer_bandwidth
migration_score = migration_benefit - migration_cost
```

当 `migration_score > threshold` 时，触发迁移操作。

#### 2.2.3 迁移流程

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  热度评估    │────▶│  决策引擎    │────▶│  执行引擎    │
│  (周期触发)  │     │  (优先级排序)│     │  (异步传输)  │
└──────────────┘     └──────────────┘     └──────────────┘
       │                    │                    │
       ▼                    ▼                    ▼
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ 生成迁移建议  │     │ 计算优先级    │     │ CUDA Stream  │
│ (块列表+目标 │     │ 批量分组      │     │ 异步拷贝     │
│  层级)       │     │ 并发控制      │     │ 完成回调     │
└──────────────┘     └──────────────┘     └──────────────┘
```

#### 2.2.4 预取策略

基于马尔可夫链的访问预测：

```
P(next_block = j | current_block = i) = transition_matrix[i][j]
```

当访问块i时，预取概率最高的前N个块到当前层级。

### 2.3 无碎片分配器 (Fragmentation-Free Allocator)

#### 2.3.1 功能描述

基于PagedAttention思想，实现固定大小块的内存管理，彻底消除内存碎片。

#### 2.3.2 块设计

- **块大小**：16KB（可配置）
- **块对齐**：4KB页对齐，支持GPU直接访问
- **块结构**：
  ```
  ┌────────────────────────────────────────────┐
  │ Block Header (64B)                          │
  │  - BlockId (8B)                             │
  │  - State (1B)                               │
  │  - Tier (1B)                                │
  │  - RefCount (2B)                            │
  │  - LayerId (2B)                             │
  │  - SequencePos (8B)                         │
  │  - Reserved (42B)                           │
  ├────────────────────────────────────────────┤
  │ Data Area (16KB - 64B)                      │
  │  - K Cache (8KB - 32B)                      │
  │  - V Cache (8KB - 32B)                      │
  └────────────────────────────────────────────┘
  ```

#### 2.3.3 内存池设计

```cpp
class MemoryPool {
private:
    // VRAM池
    void* vram_pool_;
    size_t vram_capacity_;
    std::vector<BlockId> vram_free_list_;
    
    // RAM池（锁页内存）
    void* ram_pool_;
    size_t ram_capacity_;

    // SSD交换区
    int ssd_fd_;

    std::unordered_map<BlockId, BlockMetadata> block_metadata_;

```

#### 2.3.4 分配算法

采用位图+链表的混合管理策略：

1. **快速分配**：从空闲链表头部取块，O(1)时间复杂度
2. **批量分配**：预分配连续块，减少分配次数
3. **延迟释放**：释放的块加入待回收列表，批量处理

## 四、性能优化策略

### 4.1 内存优化

1. **锁页内存**：Host端使用pinned memory，加速GPU传输
2. **内存池化**：预分配内存池，避免运行时分配开销
3. **压缩存储**：可选的KV Cache压缩，减少内存占用

### 4.2 并发优化

1. **无锁数据结构**：关键路径使用原子操作和无锁队列
2. **异步传输**：CUDA Stream实现计算与传输重叠
3. **批量处理**：迁移操作批量提交，减少系统调用

### 4.3 缓存优化

1. **预取**：基于访问模式的智能预取
2. **分层缓存**：热数据在VRAM，温数据在RAM，冷数据在SSD
3. **写回策略**：延迟写入，合并更新
